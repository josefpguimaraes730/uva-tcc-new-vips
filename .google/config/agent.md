# Gemini Agent Context: Sistema Cadastra (TCC)

## 1. Contexto do Projeto
* **Objetivo do App:** Aplicativo de triagem, cadastramento e acompanhamento reativo local-first focado na gestão do funil de conversão e integração comunitária de novos membros (uGroups/Células).
* **Min SDK / Target SDK:** API 24 / API 35.
* **Linguagem Principal:** Kotlin.

## 2. Arquitetura e Padrões de Código
* **Arquitetura:** MVVM (Model-View-ViewModel) com persistência local baseada no padrão Repository, utilizando Room Database para cache local offline estruturado.
* **UI Framework:** Desenvolvimento Híbrido Reativo (Jetpack Compose encapsulado em Android Fragments acoplados via ViewPager2 na `RegistrationActivity`).
* **Gerenciamento de Estado:** Fluxos assíncronos e frios (`Flow`) expostos como estados quentes observáveis (`StateFlow`) pela ViewModel e coletados na UI via `.collectAsState()`.
* **Fluxo de Dados:** Unidirecional (UDF - Unidirectional Data Flow).
* **Injeção de Dependência:** Manual via Escopo de Atividade Compartilhada (`ViewModelProvider(requireActivity())`).
* **Concorrência:** Kotlin Coroutines (`viewModelScope.launch`) e execução de I/O em pool de threads dedicado (`Dispatchers.IO`).

## 3. Diretrizes de Desenvolvimento e Restrições
* **Tratamento de Erros:** Proibido o uso de blocos `catch` vazios. Exceções devem propagar estados de erro explícitos para a camada de visualização ou notificações visuais controladas.
* **Modularização de Telas:** Telas complexas são estruturadas de forma que o arquivo `XScreen.kt` gerencie exclusivamente acoplamentos de estados arquiteturais e Scaffold, enquanto `ComponentesX.kt` forneça funções puras de UI sem dependência direta de ViewModels.
* **Gerenciamento de Chaves de Lista:** Todo escopo de `LazyColumn` deve conter o parâmetro estável `key` mapeando atributos únicos (como `key = { it.nome }`) para otimização da árvore de recomposição.

# Especificação de Software

## Documento de Visão e Escopo
O sistema resolve o problema de descentralização e indisponibilidade de rede no acolhimento de novos membros. Ele permite o cadastro instantâneo de dados pessoais, conexões espirituais e o direcionamento assistido por algoritmos locais a uma célula regional (uGroup), guardando os dados de forma persistente offline até a próxima janela de sincronização remota.

## Requisitos Funcionais
* **RF001 - Importação de Arquivos:** O sistema deve ler e interpretar strings de arquivos CSV/TXT locais para povoamento em lote das entidades uGroup no banco de dados Room.
* **RF002 - Cadastro com Inteligência de Funil:** O formulário deve preencher automaticamente a propriedade de estágio baseado no preenchimento de campos espirituais (Novo Registro = `TRIAGEM`; Edição, Apelo ou uGroup Indicado = `ACOMPANHAMENTO`).
* **RF003 - Painel de Métricas Reativo:** O Dashboard deve ler o banco de dados Room reativamente e exibir a contagem total de participantes alocados por estágio obrigatório (`TRIAGEM`, `ACOMPANHAMENTO`, `ENCAMINHADO`).
* **RF004 - Transição de Abas Dinâmica:** Ao salvar um novo registro no formulário Compose, o fragmento correspondente deve emitir um callback de sucesso que dispara uma transição de tela direta para a aba de estatísticas.

## Modelo Lógico do Banco de Dados (Room)
1. **`tabela_celula` (`CelulaEntity`)**
    * `id`: Long (PrimaryKey, autoGeneratedValue)
    * `nome`: String
    * `lider`: String
    * `whatsappLider`: String
    * `anfitriao`: String
    * `whatsappAnfitriao`: String
    * `bairro`: String

2. **`tabela_participantes` (`ParticipanteEntity`)**
    * `idLocal`: Long (PrimaryKey, autoGeneratedValue)
    * `usuarioLocalId`: String (FK para SessaoUsuarioEntity)
    * `nome`: String
    * `telefone`: String
    * `estagioFunil`: String (`TRIAGEM`, `ACOMPANHAMENTO`, `ENCAMINHADO`)
    * `dataCriacao`: Long (Timestamp)
    * `sincronizado`: Boolean
    * `idRemoto`: String?

3. **`sessao_usuario` (`SessaoUsuarioEntity`)**
    * `usuarioId`: String (PrimaryKey)
    * `nome`: String
    * `email`: String
    * `perfil`: String
    * `dataUltimoAcesso`: Long
